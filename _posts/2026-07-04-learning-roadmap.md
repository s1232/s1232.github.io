## Learning roadmap
It is probably too passive to think of the blog as strictly a learning log. Let's instead build out a high level plan for future learning, some kind of roadmap with a directional impulse. I don't use AI for writing, which should be obvious from my clunky Norwegian-influenced writing style, but I did consult ChatGPT when coming up with the simple roadmap below.

The disussion was on the fact that I am currently spending an excessive amount of time choosing between Rust and Java as well as breaking the habit of consuming and superficially tapping into the learning of others. Since I have already some momentum with Rust I decided against switching to Java now, spending even more time restarting. There are real reasons to approach Java over Rust, its massive adoption being the prominent one and it is exiciting eveloution from Project Valhalla another one. But the Java ecosystem will have to come later if still considered useful.

So the starting point is **Rust, open source and data engineering or integration software projects.**

1. Become comfortable reading the internals of Apache DataFusion.
2. Learn the Apache Arrow memory model and the Apache Parquet format well enough to explain them to someone else.
3. Make several small documentation, test, or bug-fix contributions to DataFusion.
4. Build a small SQL engine or Parquet analysis tool in Rust to deepen your understanding.

There it is, my learning roadmap over the coming six months, July through December 2026. I will close out this blog post now to get to it.
